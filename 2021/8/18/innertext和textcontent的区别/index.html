innerText和innerHTML的区别自不必说，但innerText和textContent的区别还是要扒一扒的。

# 区别

  • textContent 会获取所有元素的内容，包括·<script> 和 <style>元素，然而 innerText 只展示给人看的元素。

  • textContent会返回节点中的每一个元素。相反，innerText受 CSS 样式的影响，并且不会返回隐藏元素的文本

    • 此外，由于 innerText 受 CSS 样式的影响，它会触发回流（ reflow (opens new window) ）去确保是最新的计算样式。（回流在计算上可能会非常昂贵，因此应尽可能避免。）

  • 与 textContent 不同的是, 在 Internet Explorer (小于和等于 11 的版本) 中对 innerText 进行修改， 不仅会移除当前元素的子节点，而且还会永久性地破坏所有后代文本节点。在之后不可能再次将节点再次插入到任何其他元素或同一元素中。

  • 兼容性innerText IE6就开始支持，而textContent IE9浏览器才开始支持

# 总结

innerText由于存在诸多特别的特性、以及兼容性差异，以及性能方面问题，以及实际开发的需求的考量，不推荐使用，推荐使用textContent获取文本内容。

var text = dom.textContent;

如果你的项目还需要兼容IE8浏览器，则使用下面的代码：

var text = dom.textContent || dom.innerText;

其实前端的东西都是在慢慢发展完善的，跟数码产品一样，类似的产品肯定是越新的越完善，所以只要查一下caniuse (opens new window)网站用兼容性差的那个就行了，时间不够甚至都不用去深入细节，除非你要兼容被淘汰的浏览器。
