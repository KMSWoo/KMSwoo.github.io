最近在写Vue组件的时候遇到一个组件复用时行为很奇怪的问题。

# 问题再现

最近在写一个音乐WebApp，复用一个音乐列表的时候发现行为很诡异，搜索了资料发现官网有解释。

# 官方解释

当使用路由参数时，例如从 /user/foo 导航到 /user/bar，原来的组件实例会被复用。因为两个路由都渲染同个组件，比起销毁再创建，复用则显得更加高效。不过，这也意味着组件的生命周期钩子不会再被调用。

# 解决方案

复用组件时，想对路由参数的变化作出响应的话，你可以简单地 watch (监测变化) $route 对象：

const User = {
  template: '...',
  watch: {
    $route(to, from) {
      // 对路由变化作出响应...
    }
  }
}

或者使用 2.2 中引入的 beforeRouteUpdate 导航守卫 (opens new window)：

const User = {
  template: '...',
  beforeRouteUpdate(to, from, next) {
    // react to route changes...
    // don't forget to call next()
  }
}

# 原理

本质就是要阻止组件被框架因为节省开销的机制而复用，v-for的key设计也是这个原因。

所以最简单的方法就是：

<router-view :key="new Date().getTime()"></router-view>
