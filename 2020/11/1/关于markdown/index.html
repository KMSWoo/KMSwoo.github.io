Markdown是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档，可以导出为HTML 、Word、图像、PDF、Epub 等多种格式的文档。

# 发展历史

# 创建

创始人约翰·格鲁伯 (opens new window)在2004年创造了Markdown语言，在语法上有很大一部分是跟亚伦·斯沃茨 (opens new window)共同合作的。

# 标准化

随着时间的推移，出现了许多Markdown实现，越来越偏离初衷和模糊不清，急需实现标准化，但创始人反对完全的标准化。

从2012年开始，标准化工作如火如荼的进行。

2014年9月，格鲁伯反对在这一工作中继续使用“Markdown”这个名字，其被更名为CommonMark。并发布了规范、参考实现和测试包的几个版本，计划在2018年宣布最终的1.0规范和测试包。

2016年3月发布了RFC 7763和RFC 7764。RFC 7763 从原始变体引入了MIME类型 text/markdown。

# GFM

2017年，GitHub发布了基于CommonMark的GitHub Flavored Markdown（GFM (opens new window)）的正式规范。除了表格、删除线、自动链接和任务列表被GitHub规范作为扩展添加之外，它遵循CommonMark规范。

# 基本语法

这些是 John Gruber 的原始设计文档中列出的元素。所有 Markdown 应用程序都支持这些元素。

元素 Markdown 语法
标题（Heading） (opens new window) # H1## H2### H3
粗体（Bold） (opens new window) **bold text**
斜体（Italic） (opens new window) *italicized text*
引用块（Blockquote） (opens new window) > blockquote
有序列表（Ordered List） (opens new window) 1. First item2. Second item3. Third item
无序列表（Unordered List） (opens new window) - First item- Second item- Third item
代码（Code） (opens new window) code
分隔线（Horizontal Rule） (opens new window) ---
链接（Link） (opens new window) [title](https://www.example.com)
图片（Image） (opens new window) ![alt text](image.jpg)

# 扩展语法

这些元素通过添加额外的功能扩展了基本语法。但是，并非所有 Markdown 应用程序都支持这些元素。

元素 Markdown 语法
表格（Table） (opens new window) | Syntax | Description || ----------- | ----------- || Header | Title || Paragraph | Text |
代码块（Fenced Code Block） (opens new window) { "firstName": "John", "lastName": "Smith", "age": 25}
脚注（Footnote） (opens new window) Here's a sentence with a footnote. [^1][^1]: This is the footnote.
标题编号（Heading ID） (opens new window) ### My Great Heading {#custom-id}
定义列表（Definition List） (opens new window) term: definition
删除线（Strikethrough） (opens new window) ~~The world is flat.~~
任务列表（Task List） (opens new window) - [x] Write the press release- [ ] Update the website- [ ] Contact the media
